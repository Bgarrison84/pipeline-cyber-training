# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-05-17
**Phases:** 1–8
**Total Plans:** 30

## Overview

A static GitHub Pages web application that teaches IT/OT admins at oil and gas pipeline companies the PowerShell skills and cybersecurity concepts needed to meet TSA Pipeline Security Directive and NIST SP 800-82 Rev 3 compliance requirements. Shipped all five learning modules (Logging & Auditing, Network Hardening, Account & Access Management, Incident Response, Patch Management) with guided lessons, quizzes, a simulated PowerShell terminal, and scenario-based exercises — 26/26 requirements met across 8 phases in 7 days.

## Milestone Summary

**Key Accomplishments:**
1. Full five-module training platform shipped as a static GitHub Pages site — hash routing, Vite build, zero-server architecture
2. Custom vanilla JS simulated PowerShell terminal with regex-match validation, contextual hints, and always-visible SIMULATOR label
3. Branching scenario engine, searchable compliance control index (14 controls), and printable training log artifact
4. All five modules fully populated with lessons, quizzes, exercises, and scenarios — 26/26 requirements met in 7 days
5. OT/ICS patching explicitly distinguished from IT patching throughout (air-gapped, vendor qualification, risk-based deferral)
6. 167 automated tests green at completion; all TSA version strings sourced from compliance-refs.json — zero hardcoded version strings

**Key Decisions:**

| Decision | Rationale |
|----------|-----------|
| Vanilla JS + Vite + Tailwind CSS v4 | No framework needed for bounded content site; Tailwind v4 requires build step |
| Hash-based routing (#/module/:id) | Eliminates GitHub Pages SPA 404 problem; no server config needed |
| Custom vanilla JS terminal (not jQuery Terminal) | Simpler, no dependency, sufficient for whitelist-only fake-terminal pattern |
| localStorage for progress | Only viable option for no-auth static site; wrapped in single progressStore.js |
| Hybrid JSON + Markdown content | JSON for metadata/quiz/exercises; Markdown for lesson bodies (human-editable in GitHub browser) |
| data/compliance-refs.json for all version strings | TSA directive version strings must never be hardcoded; one-line edit on revision |
| schemaVersion: 1 in localStorage from day one | Migration runner depends on this key always being present |
| Wave 0 stub file pattern (Phase 5) | Create stub files that throw Error('not implemented') — Vite resolves paths at transform time |
| Dynamic import('./sidebar.js') | Breaks the quiz-engine ↔ sidebar circular dependency at module evaluation time |
| vi.hoisted() for Vitest mock variable captures | Lifts variable into hoisting zone alongside mock factory when static imports exist |

**Issues Resolved:**
- Sidebar `<a>` tag not closed — caused all 5 modules not to render (fixed in Phase 7)
- Kebab-case Lucide icon names required (BookOpen, AlertTriangle) — icon case sensitivity (fixed in Phase 7/8)
- Happy-dom does not define `window.print` — added `tests/setup.js` as Vitest setupFiles (Phase 6)
- Vite import-analysis static resolution failure — stub file pattern with `throw Error('not implemented')` (Phase 5)

**Technical Debt Incurred:**
- `ECONNREFUSED` message in npm test output (pre-existing unrelated network check, not affecting test run)
- Compliance SME review of content not yet scheduled (identified during Phase 7)
- TSA SD-02F expires May 2, 2026 — update needed in compliance-refs.json
