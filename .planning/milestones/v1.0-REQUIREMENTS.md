## v1 Requirements (All Complete)

### App Shell

- [x] **SHELL-01**: User can navigate between all 5 modules and their lessons via a persistent sidebar or top navigation menu
  - *Outcome: Validated — sidebar with hash navigation, all 5 modules visible*
- [x] **SHELL-02**: Every module and lesson has a unique, linkable URL using hash-based routing (e.g. `#/module/logging-auditing/lesson/1`)
  - *Outcome: Validated — hash router with bookmark-safe URLs*
- [x] **SHELL-03**: Visual progress bars show completion state at both lesson and module level
  - *Outcome: Validated — 4px sidebar progress bars + module-view lesson status badges*
- [x] **SHELL-04**: Compliance index page maps control IDs (TSA SD-02F, NIST SP 800-82 Rev 3) to every lesson and exercise that covers them
  - *Outcome: Validated — compliance-index.json with 14 controls, searchable index page*

### Content & Lessons

- [x] **CONT-01**: Lessons are authored in Markdown files and rendered in-browser without a build step
  - *Outcome: Validated — marked.js v17 fetch pipeline*
- [x] **CONT-02**: PowerShell code blocks render with full Shiki syntax highlighting (PS 5.1 grammar)
  - *Outcome: Validated — Shiki v4 with PS 5.1 grammar*
- [x] **CONT-03**: Every code block has a one-click copy-to-clipboard button
  - *Outcome: Validated — clipboard handler in lesson-view.js*
- [x] **CONT-04**: Each lesson displays the TSA and/or NIST control IDs it covers, sourced from a shared compliance-refs data file
  - *Outcome: Validated — compliance bar in lesson-view, IDs from frontmatter, never hardcoded*

### Simulated PowerShell Terminal

- [x] **TERM-01**: Terminal accepts expected commands per exercise and returns helpful error feedback for wrong commands
  - *Outcome: Validated — regex-match per step, exercise-view.js*
- [x] **TERM-02**: Accepted commands produce realistic-looking PowerShell output (not placeholder or lorem text)
  - *Outcome: Validated — PS 5.1 output with CategoryInfo/FullyQualifiedErrorId format*
- [x] **TERM-03**: Plausible near-miss commands (e.g. wrong parameter name) receive a contextual hint rather than generic failure
  - *Outcome: Validated — hintPatterns regex array per exercise step*
- [x] **TERM-04**: Terminal UI displays a persistent label making clear it is a simulator — commands do not run on any real system
  - *Outcome: Validated — "PS SIMULATOR — commands do not run on any real system" always visible*

### Assessment & Progress

- [x] **ASSESS-01**: Each lesson includes multiple-choice quiz questions with per-answer explanatory feedback (not just correct/incorrect)
  - *Outcome: Validated — quiz-engine.js with per-answer explanation field*
- [x] **ASSESS-02**: Scenario exercises walk learners through realistic compliance incidents with branching decision points and outcome explanations
  - *Outcome: Validated — scenario-view.js with decision-tree JSON schema*
- [x] **ASSESS-03**: LocalStorage saves the learner's exact progress; returning learners resume at the lesson and step they left
  - *Outcome: Validated — progressStore with lastVisited auto-resume*
- [x] **ASSESS-04**: Learner can generate a printable completion summary labeled as a training log artifact (explicitly not a compliance certification)
  - *Outcome: Validated — completion-summary-view.js with statutory disclaimer + @media print CSS*

## Out of Scope (Validated at v1.0)

- **Real PowerShell execution** — impossible on static host; security risk regardless
- **User accounts / authentication** — localStorage only for v1; no login, no server
- **Video content** — text, code, and interactive exercises only
- **Mobile-first layout** — desktop-primary; IT/OT admins work at workstations
- **Gamification** — research confirms this backfires with compliance-focused professional audiences
- **Social sharing of completion** — operational security risk in pipeline OT context
- **NERC CIP as mandatory framework for pipeline operators** — does not apply; electric utility standard only
