<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL hash → moduleId/exerciseId | URL params extracted by extractParams() via decodeURIComponent; used in fetch URL and href attrs, never injected into innerHTML text content |
| exercise JSON → DOM | All prose fields (title, description, context, instruction, hint) pass through esc() before innerHTML; successOutput uses textContent via terminal.appendOutput() |
| user terminal input → RegExp | Input is only passed to RegExp.test(); never reflected into DOM as innerHTML |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-W2-01 | Tampering | exercise JSON fields → innerHTML | mitigate | esc() applied to title, description, context, instruction, hint before all innerHTML assignments; acceptance criteria verifies esc(exercise.) calls |
| T-05-W2-02 | Tampering | successOutput → terminal output | mitigate | terminal.appendOutput() uses line.textContent — XSS impossible regardless of successOutput content |
| T-05-W2-03 | Tampering | moduleId/exerciseId in href | mitigate | esc() wraps moduleId/exerciseId before href attribute insertion (renderExerciseError "Return to module" link) |
| T-05-W2-04 | Tampering | malformed RegExp in exercise JSON | mitigate | new RegExp() call wrapped in try/catch in handleCommand — exception swallowed, treated as no-match |
| T-05-W2-05 | Information Disclosure | localStorage quota | accept | progressStore.saveExercise() uses existing QuotaExceededError handler from Phase 3 — no new risk |
| T-05-W2-06 | Tampering | static import of sidebar.js | mitigate | Dynamic import('./sidebar.js') used exclusively — prevents circular dep AND avoids sidebar code loading at module parse time |
</threat_model>
