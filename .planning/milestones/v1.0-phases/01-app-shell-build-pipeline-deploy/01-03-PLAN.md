<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| params.moduleId → view lookup | moduleId from hash is used as array lookup key only — module title from MODULES reaches DOM, not the raw param |
| not-found view | Must never render hash content; all text is static |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering / XSS | module-view.js — moduleId in innerHTML | mitigate | moduleId used as MODULES.find() key only; module title, description, icon from MODULES (hardcoded JS object) reach DOM. renderModuleNotFound() outputs static text only — never `${moduleId}`. |
| T-03-02 | Tampering / XSS | not-found-view.js — hash content | mitigate | renderNotFound() is a fully static template literal — no dynamic values, no hash-derived strings |
| T-03-03 | Information Disclosure | compliance badge version strings | accept | Badges render compliance document version strings (public information); no secrets; same-origin JSON fetch |
</threat_model>
