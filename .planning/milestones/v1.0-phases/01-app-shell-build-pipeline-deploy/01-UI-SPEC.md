## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, badge inner padding |
| sm | 8px | Button inner padding (vertical), list item gap |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Section padding, sidebar item padding |
| xl | 32px | Layout gaps between sidebar and content |
| 2xl | 48px | Major section breaks in module placeholder views |
| 3xl | 64px | Page-level vertical padding |

Exceptions:

- Sidebar collapsed width: 48px (icon-only mode — touch-target minimum)
- Sidebar expanded width: 256px (fixed, not a spacing token — set in layout CSS)
- Top bar height: 48px (fixed)
- Sidebar toggle button hit target: 44px × 44px minimum (accessibility)

Source: CONTEXT.md D-05, D-06; desktop-primary note in key decisions block.

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Mono (code/hashes/IDs) | 13px | 400 (regular) | 1.5 | JetBrains Mono |
| Body | 14px | 400 (regular) | 1.6 | Inter |
| Heading (module title) | 20px | 600 (semibold) | 1.2 | Inter |
| Display (app name / page title) | 28px | 600 (semibold) | 1.1 | Inter |

Declared weights: exactly 2 — 400 (regular) and 600 (semibold).

Notes:

- Labels (compliance coverage label, section card labels, progress readout): rendered at Body 14px / 600 (semibold) / uppercase / letter-spacing 0.08em. The weight + case treatment differentiates labels from body text without a separate size token.
- Compliance badge IDs and small text elements use `--text-mono` (13px / JetBrains Mono / 400).
- "OT Security Lab" app display name in top bar: 28px / 600 (semibold) / `--color-accent`. The accent orange at 28px creates strong visual differentiation without requiring a third weight.
- Sidebar lesson labels (grayed out items): Body 14px / 400 / opacity 40% — communicates disabled without a separate color token.
- All font sizes are declared as Tailwind v4 CSS custom properties in `@theme` block so downstream phases inherit them.

Source: CONTEXT.md D-10 (utilitarian typography), D-09 (internal tool feel). Defaults applied from standard scale.

## Tailwind v4 Token Declaration

All tokens MUST be declared as CSS custom properties in the `@theme` block of `src/style.css`. This enables forking/rebranding per CONTEXT.md D-12.

```css

  --color-bg-base: #1a1a1a;
  --color-bg-secondary: #2a2a2a;

  --color-accent: #f97316;

  --color-text-primary: #f5f5f4;
  --color-text-muted: #737373;
  --color-text-on-light: #1a1a1a;
  --color-border: #3a3a3a;
  --color-destructive: #ef4444;

  /* Badge colors */
  --color-badge-tsa-bg: #1e3a5f;
  --color-badge-tsa-text: #93c5fd;

  --font-sans: "Inter", system-ui, sans-serif;
  --font-mono: "JetBrains Mono", "Fira Code", monospace;
  --text-mono: 0.8125rem;    /* 13px — monospace: code, hashes, compliance IDs */
  --text-body: 0.875rem;     /* 14px — body text and labels (labels use semibold+uppercase) */
  --text-heading: 1.25rem;   /* 20px */
  --text-display: 1.75rem;   /* 28px */

  /* Spacing (mirrors 8-point scale) */
  --spacing-xs: 0.25rem;   /* 4px */

  --spacing-md: 1rem;      /* 16px */

  /* Layout (not spacing tokens — fixed dimensions) */
  --sidebar-width-expanded: 256px;
  --sidebar-width-collapsed: 48px;
  --topbar-height: 48px;
}
```

## Component Inventory (Phase 1 Only)

These are the UI components the executor must build for Phase 1. No pre-built library — all hand-coded with Tailwind v4 utilities.

### 1. App Shell (`index.html` + CSS layout)

- Outer grid: `display: grid; grid-template-columns: var(--sidebar-width-expanded) 1fr; grid-template-rows: var(--topbar-height) 1fr; height: 100vh`
- When sidebar collapsed: column 1 switches to `var(--sidebar-width-collapsed)` via `.sidebar--collapsed` class toggle
- Background: `--color-bg-base`
- Sidebar and top bar transition: `width 200ms ease, opacity 200ms ease` — simple, no spring physics

### 2. Top Bar

- Height: 48px, background `--color-bg-secondary`, border-bottom `1px solid --color-border`
- Left: "OT Security Lab" in Display type (28px, weight 600 semibold, `--color-accent`). The app name itself is the accent — it is the primary visual anchor.
- Right: two items — "Compliance Index" link (14px, semibold, muted text, hover → accent) + overall progress readout ("0 / 5 modules" at 14px semibold uppercase letter-spacing — placeholder text in Phase 1)
- Top bar does NOT scroll — `position: sticky; top: 0; z-index: 50`

### 3. Sidebar

- Background: `--color-bg-secondary`, border-right `1px solid --color-border`
- Module items: 24px vertical padding, full width, cursor pointer
- Active module: left border 3px `--color-accent`, background `rgba(249, 115, 22, 0.08)` (accent at 8% opacity)
- Active module text: `--color-accent`
- Inactive module text: `--color-text-primary`
- Chevron icon (Lucide ChevronDown / ChevronRight): 16px, muted, rotates 90° when module expands — CSS transform, no JS animation library
- Lesson subitems (visible but disabled in Phase 1):
  - Indented 32px from module left
  - Text: 14px / 400 / `--color-text-muted` / opacity 40%
  - No hover state, no cursor pointer — `pointer-events: none`
  - Label: exact lesson title as it will appear in Phase 2 (e.g., "Introduction to Windows Event Logs")
- Collapse toggle: chevron-left icon button, 44px × 44px hit target, positioned at bottom of sidebar. On collapse, all labels hide — only module icons remain (Lucide icons, 20px).
- Module icons (Lucide): BookOpen (Logging), Shield (Network), Users (Account), AlertTriangle (Incident), Wrench (Patch)

### 4. Module Placeholder View

Rendered in the `#app` content area when a module is navigated to.

Structure (top to bottom):
1. Module title: Heading (20px / semibold) + module icon (24px, accent color) inline
2. Goal statement: Body text (14px / 400) in a `--color-bg-secondary` callout block with left border 3px `--color-border`
3. Compliance IDs row: Horizontal list of badge components (TSA / NIST badges per module's coverage — sourced from `data/compliance-refs.json`)
4. Four labeled section skeletons, each a card:
   - Card background: `--color-bg-secondary`, border `1px solid --color-border`, padding `--spacing-lg`
   - Section label: 14px / 600 (semibold) / `--color-text-muted` / uppercase / letter-spacing 0.08em
   - Section names: "Lessons", "Quizzes", "Terminal Exercises", "Scenarios"
   - Content of each card: a single line of muted italic text — see Copywriting section
5. All four section cards are arranged in a 2×2 grid on desktop (min-width 1024px), stacked on narrow viewports

### 5. Compliance Badge

- Inline component: rounded pill shape, 4px border-radius, padding `4px 8px`
- TSA badge: background `--color-badge-tsa-bg`, text `--color-badge-tsa-text`, monospace 13px (`--text-mono`)
- NIST badge: background `--color-badge-nist-bg`, text `--color-badge-nist-text`, monospace 13px (`--text-mono`)
- Text content: the control ID string exactly (e.g., "TSA SD-02F", "NIST SP 800-82 Rev 3")
- Never render version strings as inline text — always route through `compliance-refs.json`

### 6. Home View (Module Catalog)

Rendered at `#/` route — lists all 5 modules as cards.

- Card: `--color-bg-secondary`, border `1px solid --color-border`, padding `--spacing-lg`, full-width in a vertical list (desktop: max-width 800px, centered)
- Module name: Heading size (20px / semibold)
- Module description: Body (14px / 400 / muted)
- Estimated time (if available): 14px / semibold / uppercase / letter-spacing 0.08em / muted
- Compliance badges row below description
- Arrow-right icon (Lucide ArrowRight, 16px, muted) — indicates navigability
- Hover state: border color → `--color-accent` (transition 150ms), no background change

## Interaction Contracts

### Sidebar Toggle

- Toggle button: bottom of sidebar, always visible (not hidden when collapsed)
- Collapsed state: sidebar width animates to 48px, all text labels hidden (`opacity: 0; width: 0; overflow: hidden`), only module icons remain visible with 44px hit targets
- Expanded state: sidebar width animates to 256px, labels fade in
- CSS transition: `width 200ms ease`, text `opacity 150ms ease 50ms` (slight delay so text doesn't appear before sidebar is wide enough)
- Toggle does NOT affect top bar or content area width — content area fills remaining viewport

### Module Expand/Collapse (Sidebar)

- Click on inactive module: collapses previously active module, expands clicked module, updates URL hash to `#/module/{id}`
- Click on already-active module: navigates to `#/module/{id}` (re-renders view, no collapse)
- Lesson subitems slide in with `max-height` transition: `0 → auto`, `200ms ease`
- Only one module is expanded at a time

### Hash Navigation

- All `<a>` tags use `href="#/module/{id}"` or `href="#/lesson/{moduleId}/{lessonId}"`
- No JavaScript `history.pushState` — hash changes are native history entries
- On `hashchange` event: router matches route, tears down current view, renders new view into `#app`
- Back/Forward browser buttons work correctly (native hash history)
- On page load with no hash (`#/` or empty): render Home View

### Disabled Lesson Items

- Rendered as `<span>` (not `<a>` or `<button>`) — semantically non-interactive
- `pointer-events: none`, `cursor: default`, opacity 40%
- No tooltip or hover state in Phase 1
- These items are VISIBLE so the learner sees the module's structure — they are not hidden

### Focus Management

- All interactive elements have a visible focus ring: `outline: 3px solid #f97316; outline-offset: 2px`
- Tab order follows DOM order: top bar → sidebar modules → sidebar toggle → main content
- Sidebar toggle is reachable by keyboard
- No focus traps in Phase 1 (no modals or overlays)

## Accessibility Contract

- Semantic HTML required: `<header>`, `<nav>`, `<main>`, `<section>`, `<article>` — no `<div>` soup
- Sidebar `<nav>` has `aria-label="Module navigation"`
- Active module item: `aria-current="page"` on the containing `<a>` or `<button>`
- Disabled lesson items: `aria-disabled="true"` + `aria-label="{lesson title} — available in Phase 2"`
- Sidebar toggle button: `aria-label="Collapse navigation"` / `aria-label="Expand navigation"` (dynamic)
- In collapsed mode, each sidebar module link includes `aria-label='{module name}'` (e.g., `aria-label='Logging & Auditing'`) so screen readers announce the destination when icons-only are visible.
- All color contrast passes WCAG AA:
  - Body text `#f5f5f4` on `#1a1a1a` background: contrast ratio ≈ 17:1 (AAA)
  - Accent `#f97316` on `#1a1a1a` background: contrast ratio ≈ 4.8:1 (AA pass for normal text)
  - Muted text `#737373` on `#2a2a2a` background: contrast ratio ≈ 4.6:1 (AA pass at 14px)
  - Badge text `#93c5fd` on `#1e3a5f`: contrast ratio ≈ 5.2:1 (AA)
  - Badge text `#86efac` on `#1a3a2a`: contrast ratio ≈ 5.4:1 (AA)
- Keyboard navigation: all interactive elements reachable by Tab in logical order
- Focus ring: `3px solid #f97316`, `outline-offset: 2px` — never suppressed
