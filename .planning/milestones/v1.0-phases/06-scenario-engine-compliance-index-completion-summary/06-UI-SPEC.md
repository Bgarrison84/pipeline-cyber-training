## Spacing Scale

Declared values (all multiples of 4):

| Token | CSS Variable | Value | Usage in Phase 6 |
|-------|-------------|-------|------------------|
| xs | `var(--spacing-xs)` | 4px | Icon gaps, badge gaps, option button margins |
| sm | `var(--spacing-sm)` | 8px | Phase node card internal padding, outcome block padding |
| md | `var(--spacing-md)` | 16px | Card padding, section padding, compliance table cell padding |
| lg | `var(--spacing-lg)` | 24px | Between scenario header and first phase node, between control sections |
| xl | `var(--spacing-xl)` | 32px | View wrapper padding, between print summary sections |
| 2xl | `var(--spacing-2xl)` | 48px | Between major sections in completion summary |
| 3xl | `var(--spacing-3xl)` | 64px | Not used in Phase 6 |

Exceptions:
- Scenario option buttons: min-height 44px (accessibility touch target — same as terminal input in Phase 5)
- Completion summary print layout: 32px page margin on all sides (CSS @media print)
- Compliance index control section vertical rhythm: 24px between each control ID group

Source: `src/style.css` @theme spacing tokens. All values carry forward from 05-UI-SPEC.md.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1a1a1a` (`var(--color-bg-base)`) | App background, all three view wrappers |

| TSA badge text | `#93c5fd` (`var(--color-badge-tsa-text)`) | TSA badge label text |
| NIST badge bg | `#1a3a2a` (`var(--color-badge-nist-bg)`) | NIST control ID badge background |
| NIST badge text | `#86efac` (`var(--color-badge-nist-text)`) | NIST badge label text |
| Muted text | `#737373` (`var(--color-text-muted)`) | Locked phase node instructions, compliance index item type labels ("lesson", "exercise", "scenario"), completion summary meta rows |
| Border | `#3a3a3a` (`var(--color-border)`) | All card borders, section dividers |

Accent (`#f97316`) reserved for: active scenario phase node (left-border, icon, counter text), scenario link button in module-view ("Start Scenario →"), "Continue" button in scenario phase progression. Never used for body text, locked states, or static labels.

Correct green (`#22c55e`) reserved for: correct option visual feedback after reveal, phase node completion state, scenario completion banner. Reuses existing value from quiz-engine.js and exercise-view.js — not a new token.

Source: `src/style.css` @theme, `src/quiz-engine.js` (green `#22c55e`), `src/badge.js` (badge colors). All values are existing tokens; no new color tokens needed in Phase 6.

## Interaction & States

### I-01: Scenario Phase Progression

State machine for each visit (fresh mode):

1. **Initial render:** Show scenario header (C-02) + first phase node (C-03) in active state. All subsequent phase nodes: not rendered (injected lazily as user progresses).
2. **User clicks option button (C-04):** Lock all option buttons in current phase (pointer-events none). Apply correct/incorrect visual states. Inject outcome block below options. Inject "Continue →" button below outcome block. Show outcome text immediately — no animation delay.
3. **User clicks "Continue →":** Inject next phase node card (C-03 active state). Scroll new card into view (`scrollIntoView({ behavior: 'smooth', block: 'start' })`). Activate icons. If current phase `isFinal: true`: skip next phase injection, call `completeScenario()` instead.
4. **`completeScenario()`:** Call `progressStore.saveScenario(moduleId, scenarioId)`. Dynamic import sidebar.js then call `refreshSidebarProgress(moduleId)`. Inject C-05 completion banner. Remove or disable "Continue →" button.

Re-visit mode (scenario already complete):
- Render all phases as locked cards immediately (re-visit locked state in C-03)
- Show C-05 completion banner immediately with re-visit heading variant
- No event handlers attached

### I-02: Scenario Option Selection Feedback

- Correct option (option.correct === true): border → #22c55e, background → rgba(34,197,94,0.06), check-circle icon shown at right
- Incorrect options (option.correct === false, not picked): border → var(--color-border), opacity 0.5
- Incorrect option (picked by user): border → var(--color-destructive), opacity 0.7, x-circle icon shown at right
- All options: pointer-events none, cursor not-allowed after any pick
- activateIcons() called after DOM mutation

### I-03: Compliance Index Page Load

- Show loading skeleton (C-13) immediately
- Fetch `{BASE_URL}data/compliance-index.json`
- On success: render C-10 header + C-11 sections for each control
- On fail (non-ok or network error): render C-12 empty state
- activateIcons() after rendering

### I-04: Completion Summary Interactions

- **Name input:** Input event listener updates a JS variable `learnerName`. Value used only for rendering the name row in print output. Not persisted.
- **Print button click:** `window.print()` — no UI state changes before or after (RESEARCH.md Pitfall 2). Button remains enabled and visible on screen.
- **Print trigger:** `@media print` CSS hides sidebar, nav, print button, name input, and `.print-hide` elements. Summary column expands to 100% width. Background → white, text → black.

### I-05: Focus Ring

- All interactive elements (option buttons, Continue button, print button, name input, compliance index links) use the global `*:focus-visible` rule: `outline: 3px solid #f97316; outline-offset: 2px`
- No overrides within Phase 6 components

### I-06: Storage Unavailability Warning

- Pattern mirrors exercise-view.js (I-09 in 05-UI-SPEC.md)
- If `!progressStore.isStorageAvailable()`: prepend warning paragraph to scenario view column
- Copy: "Progress cannot be saved — storage unavailable. Your scenario completion will not be recorded."
- Style: var(--text-body), var(--color-text-muted), role="alert"
- Not applicable to compliance index view (read-only)
- On completion summary: show warning banner if storage unavailable, body "No saved progress found — storage is unavailable. Progress data cannot be displayed."

## Security Constraints (carry-forward + Phase 6 additions)

- `esc()` from `src/utils/escape.js` applied to ALL scenario JSON strings before `innerHTML` insertion: `title`, `narrative`, `phase.title`, `phase.prompt`, `option.text`, `option.outcome`
- `esc()` applied to ALL compliance-index.json strings before `innerHTML` insertion: `control.label`, `item.title`
- `esc()` applied to learner name input value before any innerHTML injection
- `safePath()` applied to `moduleId` and `scenarioId` URL segments before fetch URL construction (same allowlist `/^[a-zA-Z0-9_-]+$/` as exercise-view.js)
- `safePath()` applied to compliance-index item `moduleId` and `contentId` before building hash href strings
- `moduleId` and `scenarioId` from URL hash never injected into innerHTML text — only into safe attribute positions (`href`, `data-*`) after `esc()` wrapping
- NERC CIP references in scenario content must include disclaimer: "NERC CIP governs electric utilities; pipeline operators follow TSA directives — referenced here as a maturity benchmark."
- TSA version string ("SD-02F") never hardcoded in scenario JSON or compliance-index.json labels — all labels source from `compliance-refs.json` via `renderBadge()` or equivalent lookup
- All localStorage access exclusively through `progressStore` — no direct localStorage calls in scenario-view.js, compliance-index-view.js, or completion-summary-view.js
- Dynamic import of sidebar.js in completeScenario() — no static import (circular dep prevention)
