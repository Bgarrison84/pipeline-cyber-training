<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage read | Any string may be stored (corrupted, manually edited, injected by another script) |
| File import (user-selected JSON) | Completely untrusted — user or attacker may supply any file content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | importProgress() — JSON.parse on user file | mitigate | Wrap JSON.parse in try/catch; validate typeof parsed.schemaVersion === 'number'; check at least one known top-level key present; reject on any validation failure without overwriting _store |
| T-03-02 | Tampering | XSS via stored values rendered to innerHTML | mitigate | consumers (lesson-view.js, sidebar.js) must apply esc() to any stored string inserted into innerHTML; progress-store.js itself never writes to innerHTML |
| T-03-03 | Denial of Service | importProgress — oversized JSON file | accept | Progress data is training state only; no PII; a learner importing their own file; DoS impact is self-inflicted and limited to their own session. File size check omitted (YAGNI at this scope). |
| T-03-04 | Tampering | Prototype pollution via __proto__ in JSON | mitigate | JSON.parse is safe by default — does not assign to prototype chain; no Object.assign(target, parsed) pattern used; _store fields accessed by explicit known keys only |
| T-03-05 | Information Disclosure | localStorage contents readable by any same-origin script | accept | All data is non-sensitive training state (lesson IDs, quiz scores, timestamps); no PII, no credentials, no compliance-sensitive data stored; single-origin static site |
</threat_model>
