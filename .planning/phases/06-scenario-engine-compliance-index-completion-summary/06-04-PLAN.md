<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Learner name input → innerHTML | User types a name; must not be injected via innerHTML |
| progressStore data → HTML | Read-only progress data from localStorage; all string values esc()-protected |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-W3-01 | Tampering | Learner name in completion summary | mitigate | Name stored in JS variable only, displayed via textContent not innerHTML — no esc() needed since textContent never interprets HTML; if any innerHTML path exists, esc() is applied |
| T-06-W3-02 | Tampering | progressStore data (quiz scores, module titles) in innerHTML | mitigate | esc() applied to all string values before innerHTML injection; module.title from MODULES is developer-controlled, but esc() applied defensively |
| T-06-W3-03 | Tampering | @media print CSS scope | accept | @media print block targets element selectors only (#sidebar, #nav, .print-hide, body) — no user-controlled values in CSS |
| T-06-W3-04 | Information Disclosure | window.print() exposes all visible DOM to print | mitigate | .print-hide class applied to name input container and print button; @media print hides #sidebar and #nav; only the summary content column is visible in print output |
| T-06-W3-05 | Denial of Service | window.print() called in a loop | accept | Print button has no debounce; opening a system dialog is a user-intentional action requiring explicit confirmation; no loop risk |
</threat_model>
