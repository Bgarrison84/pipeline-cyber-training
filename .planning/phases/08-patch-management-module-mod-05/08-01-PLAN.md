H2 "Why OT Patching Is Different" — explain that OT systems (PLCs, RTUs, DCS controllers, historian servers like PIPELINE-HIST01) cannot follow IT patch cadences; three structural differences: (1) OEM qualification requirement — a patch for Windows Server running on a historian must be tested and approved by the historian OEM before it can be safely applied; applying an unapproved patch can corrupt proprietary OT software or OPC server communication; (2) 3-12 month vendor approval cycles — OT vendors typically take 3 to 12 months to qualify a security patch; this is not negligence but reflects the testing required for safety-critical OT software dependencies; (3) uptime requirements — most pipeline OT systems have planned maintenance windows measured in weeks or months, not days; a patch cannot be applied outside a scheduled window without change management approval

H2 "The OT Patch Approval Workflow" — numbered steps: (1) CISA or vendor advisory published for a relevant CVE; (2) organization assesses whether its specific OT asset version is affected; (3) OEM vendor contacted — request patch qualification status and ETA; (4) if patch is not yet qualified, document compensating controls (network isolation, enhanced monitoring) with supervisor sign-off; (5) when OEM qualifies the patch, schedule deployment in the next available maintenance window; (6) deploy offline using approved media (USB or internal media server — never direct internet connection to OT system); (7) document: patch ID, OEM qualification reference, date applied, change window ID, approver; explain that this workflow is required by the current TSA pipeline security directive for OT/ICS environments — steps cannot be skipped even if the CVE severity is critical

H2 "Offline Media Staging" — explain that most OT systems on the 10.0.0.0/24 air-gapped segment have no internet connectivity; patches must be staged on approved removable media (USB drives approved via the organization's media control policy) or an internal media server in the DMZ; include PS 5.1 code block:
