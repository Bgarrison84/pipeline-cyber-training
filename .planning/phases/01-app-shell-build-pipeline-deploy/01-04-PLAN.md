## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → GitHub Pages | CI/CD pipeline; npm ci ensures deterministic install from locked package-lock.json |
| Deployed static site → browser | All content is static; no server-side processing; same-origin JSON fetch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | GitHub Actions workflow — supply chain | mitigate | `npm ci` (not `npm install`) uses package-lock.json for deterministic installs; package-lock.json is committed |
| T-04-02 | Tampering | deploy.yml — action versions | mitigate | Pinned to @v4/@v5 (verified stable versions per RESEARCH.md); not using @latest or floating versions |
| T-04-03 | Spoofing | GitHub Pages — wrong base path | mitigate | vite.config.js base='/pipeline-cyber-training/' verified before push; dist/index.html asset paths confirmed to contain this prefix |
| T-04-04 | Denial of Service | Jekyll stripping _assets chunks | mitigate | public/.nojekyll created in Plan 01; dist/.nojekyll verified present before push |
</threat_model>
