### New tokens required for Phase 9

Two new CSS custom properties must be added to the `@theme` block in `src/style.css`:

| Token | Value | Rationale |
|-------|-------|-----------|
| `--color-badge-expired-bg` | `#2a2a2a` | Matches `--color-bg-secondary` — neutral, desaturated, conveys "inactive" |
| `--color-badge-expired-text` | `#737373` | Matches `--color-text-muted` — low-contrast text signals "not current" |

**Accent reserved for:** focus rings (`*:focus-visible`), active sidebar lesson link
(`sidebar-lesson-link--active` border and text), accent hover on lesson nav buttons.
The expired badge must NOT use accent color.

**Design rationale for expired color choice:**
- Safety orange (`#f97316`) is reserved for active/action states — using it on expired
  would signal urgency rather than staleness.
- Red (`#ef4444`, `--color-destructive`) is reserved for error states. An expired directive
  is a compliance data state, not an application error.
- Gray (`#2a2a2a` bg / `#737373` text) communicates "no longer active" without false urgency,
  and reproduces as mid-gray in print (sufficient contrast against white paper).

## Expired Badge: Complete Visual Specification

### States

The badge has two states, branching on `_complianceRefs.directives[directiveKey].status`:

| State | Trigger | Visual treatment |
|-------|---------|-----------------|
| Active (default) | `status` field absent or any value other than `"expired"` | Existing color classes unchanged |
| Expired | `status === "expired"` | Gray bg + muted text + "[EXPIRED]" text suffix + `text-decoration: line-through` on the version string |

### Active badge (existing, no change)

```html
<span class="inline-block rounded px-2 py-0.5 font-mono text-[var(--text-mono)]
             bg-[var(--color-badge-tsa-bg)] text-[var(--color-badge-tsa-text)]">
  TSA SD-02F
</span>
```

### Expired badge (new)

```html
<span class="inline-block rounded px-2 py-0.5 font-mono text-[var(--text-mono)]
             bg-[var(--color-badge-expired-bg)] text-[var(--color-badge-expired-text)]"
      title="This directive has expired. Verify successor at TSA.gov."
      aria-label="TSA SD-02F — EXPIRED">
  <span style="text-decoration: line-through;">TSA SD-02F</span> [EXPIRED]
</span>
```

### Why both strikethrough AND "[EXPIRED]" text suffix

SME-02 requires a printable SME review checklist. The `@media print` rule in `src/style.css`
forces `body { background: #ffffff; color: #000000; }`. On a printed page:
- Color alone fails: gray-on-white may not distinguish from normal text in a photocopy.
- Strikethrough alone fails: subtle, may not be noticed in a scan.
- Text label alone is sufficient but strikethrough adds redundant visual signal for screen.
Both together satisfy WCAG 1.4.1 (use of color) and SME-02 print legibility.

### tooltip / title attribute

The `title` attribute provides hover context on screen:
`"This directive has expired. Verify successor at TSA.gov."`

No JS tooltip component is required — the HTML `title` attribute is sufficient for this
inline informational badge. A link to TSA.gov is NOT added directly to the badge because:
1. The badge appears inline in lesson prose, exercise headers, and the completion summary —
   adding a link disrupts those reading contexts.
2. The SME review checklist (SME-01 task) will carry the authoritative verification note.

### `aria-label` pattern

The `aria-label` replaces screen reader announcement of the raw strikethrough text:
- Without it: screen reader reads "TSA SD-02F [EXPIRED]" with possible confusion on
  the strikethrough span.
- With it: screen reader announces "TSA SD-02F — EXPIRED" as a single coherent label.

### compliance-refs.json data model addition

Badge.js reads `_complianceRefs.directives[directiveKey]`. Phase 9 adds:

```json
{
  "TSA": {
    "name": "TSA Security Directive Pipeline-2021-02F",
    "shortName": "TSA SD-02F",
    "effectiveDate": "2025-05-03",
    "expiryDate": "2026-05-02",
    "status": "expired",
    "sourceUrl": "..."
  }
}
```

The `status` field is the single trigger. `badge.js` reads
`_complianceRefs.directives[directiveKey].status === 'expired'` to branch rendering.

When a successor is confirmed at TSA.gov:
- Add a new key (e.g. `"TSA"` updated with new shortName) OR add a new directive key.
- Set `"status": "active"` (or omit field) on the successor entry.
- The expired SD-02F entry may be kept with `"status": "expired"` for historical reference.
- No lesson files need to change — all version strings read from compliance-refs.json.

### badge.js implementation contract (for executor)

`renderBadge(directiveKey)` in `src/badge.js` must be updated to:

1. Read `_complianceRefs?.directives?.[directiveKey]?.status` — default to `'active'` if absent.
2. If `status === 'expired'`:
   - Use `bg-[var(--color-badge-expired-bg)] text-[var(--color-badge-expired-text)]` classes.
   - Wrap `shortName` in `<span style="text-decoration: line-through;">{shortName}</span>`.
   - Append literal ` [EXPIRED]` after the inner span (outside the strikethrough span).
   - Add `title="This directive has expired. Verify successor at TSA.gov."`.
   - Add `aria-label="{shortName} — EXPIRED"`.
3. If `status !== 'expired'` (active, or field absent):
   - Existing rendering unchanged.

No other files change for the badge feature. The `colorClasses` lookup in badge.js remains
for active badges. A new `expiredClasses` constant is added alongside it.

### Print behavior

The existing `@media print` rule (`src/style.css` lines 294–301) sets
`background: #ffffff; color: #000000`. The expired badge will render as:
- Background: white (overridden by print rule)
- Text: black with strikethrough + "[EXPIRED]" label in parentheses

This is legible on paper without any additional print-specific CSS.

No additional `@media print` rules are required for the expired badge.
