  <behavior>
    loadForkConfig() — fetch failure cases:
    - When fetch throws (network error): returns DEFAULT_FORK_CONFIG, not null
    - When fetch returns !res.ok (404): returns DEFAULT_FORK_CONFIG, not null
    - When fetch returns ok but body is invalid JSON: returns DEFAULT_FORK_CONFIG, not null
    - When fetch returns ok but JSON missing orgName: returns DEFAULT_FORK_CONFIG, not null
    - When fetch returns ok but activeModules is not an array: returns DEFAULT_FORK_CONFIG, not null

    loadForkConfig() — success case:
    - When fetch returns valid JSON {orgName, logoPath, activeModules}: returns the parsed object
    - After success: getForkConfig() returns the loaded config (not DEFAULT_FORK_CONFIG)

    getForkConfig() — before loadForkConfig():
    - Returns DEFAULT_FORK_CONFIG (orgName 'OT Security Lab', logoPath null, 5 modules)
    - Never returns null

    applyForkBranding() — text-only (logoPath null):
    - Sets document.title to config.orgName
    - Sets top-bar span textContent to config.orgName
    - Does not set span.innerHTML

    applyForkBranding() — with logo (logoPath set):
    - Sets document.title to config.orgName
    - Sets span.innerHTML to contain an img element with src including logoPath
    - The img src includes import.meta.env.BASE_URL prefix
    - The img alt attribute equals orgName

    router.js inactive-module guard:
    - When moduleId is not in activeModules: view renderer returns string containing "not enabled for your organization"
    - When moduleId is in activeModules: view renderer calls renderModule (not the not-enabled message)

    completion-summary-view.js active filter:
    - When activeModules = ['logging-auditing'] only: progress table contains only 1 row (logging-auditing module)
    - When activeModules contains all 5: progress table contains 5 rows
  </behavior>
