## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| fork.config.json → main.js | fork config values flow into DOM via applyForkBranding(); orgName could contain HTML if JSON is tampered |
| URL hash → router.js | moduleId extracted from hash and compared against activeModules; no injection risk since it's only used in .includes() |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-03-01 | Tampering | applyForkBranding XSS via orgName | mitigate | applyForkBranding (in fork-config.js Plan 01) uses esc() for innerHTML path and textContent for text-only path — defense is at the source, not at the call site |
| T-10-03-02 | Tampering | router.js moduleId comparison | accept | moduleId from URL is compared via Array.includes() — no eval, no innerHTML, no XSS vector; worst case is a false positive/negative on the inactive guard |
| T-10-03-03 | Elevation of Privilege | inactive module URL bypass | accept | Module content is public static files; hiding inactive modules is a UX gate, not a security gate; a determined user can still fetch the raw JSON — this is documented as UX-only filtering |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan |
