<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| network → fork-config.js | fetch() retrieves fork.config.json; content is attacker-controlled on a compromised CDN or MITM |
| fork.config.json → DOM | applyForkBranding() inserts orgName into DOM; if via innerHTML without escaping, XSS risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | public/fork.config.json | accept | Static file served by GitHub Pages; no server-side execution; attacker who can modify must already have repo write access |
| T-10-02 | Information Disclosure | fork.config.json in public/ | accept | Config is intentionally public (org name, logo path, active modules are not secrets) |
| T-10-03 | Spoofing | loadForkConfig() on parse failure | mitigate | Always return DEFAULT_FORK_CONFIG on any error — never null, never partial object; validation ensures both orgName (string) and activeModules (array) exist |
| T-10-04 | Tampering | applyForkBranding() XSS via orgName | mitigate | When logoPath is set (innerHTML path): use esc(config.orgName) from utils/escape.js on the org name text node; use textContent (not innerHTML) when logoPath is null |
| T-10-05 | Tampering | applyForkBranding() XSS via logoPath | mitigate | logoPath is used only as img src attribute via string interpolation; no innerHTML injection of logoPath value itself; img src is not executable in modern browsers |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan; no slopcheck required |
</threat_model>
