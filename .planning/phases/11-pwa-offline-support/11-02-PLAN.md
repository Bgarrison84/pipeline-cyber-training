    - initOfflineIndicator() appends #offline-indicator span into #top-bar > div.flex

    Initial state:
    - When navigator.onLine is true: label text contains 'Online'; dot background is '#22c55e'
    - When navigator.onLine is false: label text contains 'Offline'; dot background contains '#d97706' (amber)

    State transitions via window events:
    - window offline event fires -> label switches to 'Offline'; dot switches to amber (#d97706)
    - window online event fires after offline -> label switches back to 'Online'; dot switches to green (#22c55e)

    Install prompt:
    - beforeinstallprompt event: e.preventDefault() is called; #install-link display changes from 'none' to 'flex'
    - appinstalled event: #install-link display returns to 'none'
    - clicking #install-link calls deferredPrompt.prompt()

    SW integration (satisfies D-04):
    - When navigator.serviceWorker is available: addEventListener is called with 'controllerchange' as first arg

    Guard:
    - When #top-bar > div.flex is absent from DOM: initOfflineIndicator() returns without throwing

    Test all cases: input -> expected output
    - initOfflineIndicator() called, #top-bar exists -> #offline-indicator injected into DOM
    - navigator.onLine = true -> label textContent includes 'Online'
    - navigator.onLine = false -> label textContent includes 'Offline'
    - window.dispatchEvent(new Event('offline')) -> label includes 'Offline'
    - window.dispatchEvent(new Event('online')) after offline -> label includes 'Online'
    - beforeinstallprompt event dispatched -> #install-link.style.display !== 'none'
    - appinstalled event dispatched -> #install-link.style.display === 'none'
    - navigator.serviceWorker.addEventListener called with 'controllerchange'
    - empty DOM (no #top-bar) -> no throw
  </behavior>
