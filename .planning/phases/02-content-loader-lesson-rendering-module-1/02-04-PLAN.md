    - Code block 3: Enable logging with Set-ItemProperty, setting EnableScriptBlockLogging to 1. Note this takes effect immediately for new PS sessions — running PS sessions are unaffected until they restart.
    - Code block 4: Verify logging is active by querying the PowerShell/Operational log for Event ID 4104. This confirms the next PS command execution will be logged.
    - OT callout: In air-gapped OT environments without a domain controller, Group Policy cannot be used to distribute these registry settings. Configure the registry changes manually on each OT workstation using the PS commands above, or use a startup script on a standalone machine. NIST CM-6 (Configuration Settings) still applies — document the registry state as a configuration baseline.
    - Note on Module Logging and Transcription Logging: briefly mention that two additional logging types exist (Module Logging via EnableModuleLogging and Transcription Logging). Script Block Logging is the highest-value control because it captures decoded payloads; enable it first.

For OT domains on air-gapped networks, export and import GPO settings via USB using `Backup-GPO` and `Restore-GPO` cmdlets on the air-gapped DC (PIPELINE-DC01).
