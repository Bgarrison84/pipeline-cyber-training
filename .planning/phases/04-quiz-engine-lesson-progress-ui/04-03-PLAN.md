<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| progressStore → sidebar DOM | Progress data from localStorage drives DOM style updates; no user input crosses this boundary |
| progressStore → module-view HTML string | Progress data from localStorage is interpolated into returned HTML string; no external string values used for badge text |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-W2-01 | Tampering | module-view.js renderModule() lesson title in HTML string | mitigate | esc() applied to lesson.title and lesson.id from MODULES config before innerHTML interpolation |
| T-04-W2-02 | Tampering | sidebar.js renderAllModuleProgress — module ID in querySelector | mitigate | CSS.escape(mod.id) used in querySelector argument; mod.id originates from MODULES config (trusted authored content), not URL hash |
| T-04-W2-03 | Tampering | module-view.js status badge text | accept | Badge text ('quiz-passed', 'visited', 'unvisited') are hard-coded string literals, not derived from any external input — esc() not needed |
| T-04-W2-04 | Information Disclosure | sidebar progress bar percentage | accept | Completion percentage is derived from learner's own localStorage progress — no third-party data, no PII disclosure risk |
</threat_model>
